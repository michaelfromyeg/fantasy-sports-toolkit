---
name: trade-check
description: Quick fairness verdict on a fantasy trade stated inline, e.g. /trade-check I give Player A, I get Player B.
---

The user will state a trade inline (what they give, what they get). Give a fast verdict.

1. Value each side on rest-of-season outlook (use the connected fantasy data provider MCP for
   usage/matchups if one is available).
2. State who wins the raw value and by how much (slight / clear / lopsided).
3. Factor in the user's roster need or category need if they mentioned one.
4. End with one word in bold -- **Accept**, **Decline**, or **Counter** -- and a one-line reason.

Keep it to a few lines. This is the gut-check version; the trade-analyzer skill is for deep dives.
